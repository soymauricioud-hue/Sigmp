# 0.15 — Matriz preliminar de fragmentación para núcleo 0.1–0.6

## Estado

E2 preliminar.

## Propósito

Identificar qué partes de la capa narrativa del SMGIP alimentan los documentos iniciales del núcleo rector 0.1–0.6.

## Regla de uso

Esta matriz no redacta los documentos. Solo define de dónde debe extraerse el contenido para evitar que la IA invente estructura o mezcle capas documentales.

---

| Documento | Fragmentos fuente principales | Función documental |
|---|---|---|
| 0.1 Documento maestro / Propuesta técnica unificada | Portada, introducción, planteamiento general, lógica rectora, arquitectura modular, cierre operativo, gobernanza, trazabilidad, sostenibilidad | Define la visión completa del SMGIP |
| 0.2 Exposición de motivos | Contexto municipal, problema público, fundamentación científica, jurídica, económica y territorial | Justifica por qué el municipio debe crear el sistema |
| 0.3 Teoría de cambio | Transición asistencial-regulatoria, control de reposición, cierre operativo, sostenibilidad, auditoría, entrada asistencial-regulada | Explica cómo el sistema transforma el problema |
| 0.4 Alcance, principios y fases | Principios rectores, límites del sistema, fase asistencial, fase regulatoria, cierre normativo, capacidades prohibidas por cierre prematuro | Define qué cubre el sistema, qué no cubre y cómo avanza |
| 0.5 Glosario maestro y taxonomía operativa | Definiciones de tutor, animal comunitario, RUAM, microchip, ESCM, QR, UCR, criador, visitante, CNVR, prealta, alta plena | Fija lenguaje obligatorio |
| 0.6 Matriz de dependencias documentales | Árbol documental, estados E0–E5, orden de escritura, reglas de desbloqueo, cierre prematuro, uso controlado de IA | Define qué documento desbloquea a otro |

## Restricción

Esta matriz solo desbloquea el núcleo 0.1–0.6. No desbloquea módulos jurídicos, clínicos, financieros, tecnológicos, territoriales ni sancionatorios.
